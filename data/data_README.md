# Data Files

This folder contains input templates, example input data, and the curated future-blind benchmark dataset released with **OFETNow-ML**.

The full curated training dataset and raw literature-extracted records are not included in this repository.

## Files

### `input_template.json`

A blank JSON template for running predictions on new polymer OFET candidates.

Input fields:

```text
entry_id
polymer_name
SMILES
Mn_Kda
N_AVE
PDI
Aspect_Ratio
Capacitance
GATE
Dielectric
Fabrication_Method
Solvent_of_Spin_Coating
Test_Environment
Type
S/D
Annealing_Temperature
```

`Mn_Kda` and `N_AVE` are both included for flexibility. Users may provide either value depending on available characterization data. The preprocessing workflow handles the feature conversion required by the released models.

For categorical variables, it is strongly recommended to use the feature names and category names provided in the feature-name files under `models/`. This helps the fitted one-hot encoder correctly recognize the input categories and improves prediction reliability.

### `example_input.json`

An example JSON input file with five polymer OFET entries for testing the prediction workflow.

### `future_blind.csv`

A curated external benchmark dataset collected from literature published after the training-data cutoff. This dataset contains 30 polymer OFET entries unseen during model training and includes the following ground-truth label columns in addition to the standard input fields:

- `max_mobility` — experimentally measured maximum mobility (cm² V⁻¹ s⁻¹)
- `C-TYPE` — experimentally determined carrier transport type (p, n, or ambipolar)
- `mobility_label` — binary mobility tier label (0 = Low, 1 = High; threshold 0.5 cm² V⁻¹ s⁻¹)

These label columns are for validation purposes only and are not used as model inputs.

### `future_blind_predictions.csv`

Prediction results generated by the released OFETNow-ML models on `future_blind.csv`, including predicted labels and class probabilities for both the mobility classifier and the carrier-type classifier.

## Input Format

JSON is the recommended input format for new predictions because it clearly preserves the field structure for each polymer OFET entry.

CSV input is also supported. The prediction workflow can read a `.csv` file and automatically convert it to the JSON list-of-records format before running inference.

Each JSON input file should contain a list of entries:

```json
[
  {
    "entry_id": "EX001",
    "polymer_name": "example_polymer",
    "SMILES": "example_smiles",
    "Mn_Kda": null,
    "N_AVE": null,
    "PDI": null,
    "Aspect_Ratio": null,
    "Capacitance": null,
    "GATE": "",
    "Dielectric": "",
    "Fabrication_Method": "",
    "Solvent_of_Spin_Coating": "",
    "Test_Environment": "",
    "Type": "",
    "S/D": "",
    "Annealing_Temperature": null
  }
]
```

Missing numerical values should be written as `null`. Missing categorical values should be written as an empty string `""` only when the value is genuinely unavailable.

## Data Scope

Included:

- Input template and example input
- Future-blind benchmark dataset (30 entries with ground-truth labels)
- Model prediction results on the benchmark dataset

Not included:

- Full training dataset
- Cross-validation splits
- Per-sample training labels
